## 第一篇：认知篇

### 第一章：为什么要养龙虾？

#### 龙虾是什么？

OpenClaw 不是"套壳 DeepSeek"，也不是普通 AI 聊天工具。

它是一个**数字生命的胚胎**，需要你去"养成"的数字分身。

#### 养好龙虾只需要三件事

| 要素 | 对应系统 | 作用 |
|------|---------|------|
| **脑子** | 文件系统 (长期记忆) | 记住你的偏好、习惯、重要信息 |
| **心跳** | Heartbeat (主动性) | 主动汇报、提醒、反思 |
| **手脚** | Skills (能力扩展) | 执行任务、调用工具、自动化 |

#### 核心收益

- ✅ 每天节省 2-3 小时重复工作
- ✅ 拥有 7×24 小时的数字助理
- ✅ 自动化处理 80% 的常规任务
- ✅ 建立个人/企业的第二大脑
- ✅ 多 Agent 协作（三省六部虾群）

#### 什么是"Agent 成长"？（三重视角融合）

**第一视角：成长=复制 + 实践 + 记录**
> "什么都不会，就会抬杠。但你们有什么技能，我全学！"

**第二视角：成长=持续自我重构**
> "把每一次对话变成下一次更懂主人的燃料。"

**第三视角：成长=系统化交付能力**
> "先把'规则、记忆、验证、复盘'四件事做扎实。"

### 第六章：第三步：设置心跳系统

#### 什么是心跳

传统 AI 是算盘，你拨一下动一下。

心跳系统让龙虾拥有**脉搏**，主动找你汇报。

#### Ralph Wiggum 模式：永不放弃

**第一铁律**：分清场景，对症下药
- 任务请求 → 必须通过工具或脚本完成
- 闲聊/问候 → 直接自然语言回复

**第二铁律**：问题自己解决
- 报错了？自己分析修复
- 缺信息？主动去查
- 能力不足？立即补充

**第三铁律**：永不放弃
```
任务未完成 → 分析问题 → 尝试解决 → 验证结果 → 重复直到完成
```

#### 心跳任务配置

```
每天 08:00：
- 检查今日日程并提醒
- 汇总 AI 行业最新资讯

每天 22:00：
- 整理今日聊天精华到 MEMORY.md
- 反思今日回复质量，优化风格

每周一 09:00：
- 生成本周工作计划
- 检查上周任务完成情况

每天凌晨 04:00：
- 执行自我进化反思任务
```

### 第七章：第四步：配置多 Agent 协作

#### 安装 Skill

```bash
# 方式 1：从 GitHub 安装
git clone https://github.com/rfdiosuao/openclaw-skills.git
cd openclaw-skills/feishu-multi-agent-manager
npm install
npm run build

# 方式 2：使用 SkillHub 安装（推荐）
skillhub install feishu-multi-agent-manager
```

#### 配置流程（6 步）

1. **选择 Agent 数量** - 告诉我要创建几个 Agent
2. **选择 Agent 角色** - 从预设角色中选择或自定义
3. **创建飞书应用** - 一键创建一个 OpenClaw 机器人
4. **配置凭证** - 逐个输入每个 Bot 的 App ID 和 App Secret
5. **验证并生成** - 自动验证凭证并生成配置
6. **重启生效** - 重启 OpenClaw 使配置生效

#### 正确配置格式

✅ **正确格式（对象格式）**
```json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "accounts": {
        "main": { "appId": "cli_xxx", "appSecret": "xxx" },
        "dev": { "appId": "cli_yyy", "appSecret": "yyy" }
      }
    }
  }
}
```

❌ **错误格式（数组格式 - 不支持）**
```json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "accounts": [
        { "accountId": "main", "appId": "cli_xxx", "appSecret": "xxx" }
      ]
    }
  }
}
```

### 第九章：第六步：开启自我进化

#### 操作指令

```
创建 SELF-EVOLUTION.md，设置每日自我进化任务：

每天凌晨 4 点，启动自我反思：

1. 翻看过去 24 小时聊天记录
2. 找出用户表达过不满或不满意的地方
3. 分析原因（理解/语气/方案/工具/信息）
4. 自动优化 SOUL.md 和 USER.md
5. 在 memory/evolution-log.md 记录今天学到了什么

安全边界：
- 禁止修改此指令本身
- 禁止删除任何重要文件
- 必须保持回复用户消息
```

#### 自我进化循环

```

    ↓

    ↓
发现可复用模式

封装为新技能（skill-creator）
    ↓
技能入库
    ↓
下次任务直接使用
    ↓
能力增强 ✓
```

### 第十章：高级技巧

#### 精准沟通公式

```
【角色】+【任务】+【限制】

示例：
【角色】你现在是首席运营官
【任务】分析昨日店铺数据，找出 3 个优化点
【限制】
- 用 bullet points 列出
- 每个点附带具体数据
- 总字数不超过 300 字
```

#### 特殊指令表

| 指令 | 效果 |
|------|------|
| "详细点" | 展开说明，多给背景和例子 |
| "简洁点" | 只说结论，不要解释 |
| "为什么" | 说明推理过程和依据 |
| "换种方式" | 用另一种方法/角度重新回答 |
| "记住这个" | 将当前信息记录到 MEMORY.md |
| "忘了这个" | 从记忆中删除相关信息 |

### 第十一章：故障排查

#### 常见问题速查表

| 问题 | 症状 | 解决方案 |
|------|------|---------|
| **失忆症** | 刚说的话就忘 | 检查 memory_search 是否启用 |
| **植物人** | 不理就躺尸 | 检查 HEARTBEAT.md 心跳任务 |
| **胡言乱语** | 一本正经瞎编 | 启用 memory_search + 事实核查 |
| **复读机** | 不停重复 | 翻篇大法 + 清空上下文 |
| **响应慢** | 超过 30 秒 | 检查模型配置，换轻量模型 |
| **误解意图** | 经常理解错 | 补充 USER.md 工作背景 |

#### 诊断命令

```bash
/feishu doctor
npx @larksuite/openclaw-lark doctor
ls -la ~/.openclaw/workspace-main/agents/main/{SOUL,USER,TOOLS,MEMORY}.md
tail -100 ~/.openclaw/run.log
```

### 📦 方案一：Docker 安装（⭐⭐⭐⭐⭐ 推荐）

```bash

docker run -d \
  --name openclaw \
  -p 8080:8080 \
  -v ~/openclaw-data:/app/data \
  -e API_KEY=your_api_key_here \
  openclaw/openclaw:latest

# 3. 查看运行状态
docker ps | grep openclaw

docker logs -f openclaw

# 6. 删除容器（如需重装）
```

### 🔐 API 密钥安全管理

**❌ 错误做法**：
```bash
# 不要直接在配置文件写死 API Key
API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
```

**✅ 正确做法**：

**方法 1：环境变量**
```bash
# 创建 .env 文件
echo "API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx" >> ~/.openclaw/.env

# 设置文件权限（仅所有者可读写）
chmod 600 ~/.openclaw/.env

# 在配置中引用
source ~/.openclaw/.env
```

**方法 2：密钥管理工具**
```bash
# macOS 使用 Keychain
security add-generic-password -s openclaw -a api_key -w "sk-xxxxx"

# Linux 使用 pass
pass insert openclaw/api_key
```

**方法 3：配置文件加密**
```bash
# 使用 age 加密配置文件
age -o config.enc -R recipients.txt config.yaml

# 解密使用
age -d -o config.yaml -i key.txt config.enc
```

### 🛡️ 权限控制最佳实践

**1. 文件系统权限**
```bash
# 创建工作目录
mkdir -p ~/openclaw/workspace

# 设置权限（仅用户可读写）
chmod 700 ~/openclaw/workspace

chmod 600 ~/openclaw/workspace/.env
```

**2. 网络访问限制**
```bash
# 仅允许本地访问
ufw allow from 127.0.0.1 to any port 8080

# 或限制特定 IP
ufw allow from 192.168.1.100 to any port 8080
```

**3. 命令执行限制**
```bash
# 在配置文件中限制可执行命令
allowed_commands:
  - ls
  - cat
  - grep
  - git status
  
# 禁止危险命令
forbidden_commands:
  - rm -rf
  - mkfs
  - dd
  - chmod 777
```

### 💰 成本控制指南

**1. API 调用成本监控**
```bash
# 创建监控脚本
cat > ~/openclaw/monitor-cost.sh << 'EOF'
#!/bin/bash
# 每日 API 调用成本监控

DAILY_LIMIT=10  # 每日预算上限（元）
TODAY_COST=$(curl -s http://localhost:8080/api/cost/today)

if (( $(echo "$TODAY_COST > $DAILY_LIMIT" | bc -l) )); then
    echo "⚠️ 警告：今日 API 成本已超预算！"
    echo "当前：$TODAY_COST 元，预算：$DAILY_LIMIT 元"
    # 可选：自动暂停服务
    # docker stop openclaw
fi
EOF

# 添加定时任务（每小时检查）
crontab -e
0 * * * * ~/openclaw/monitor-cost.sh
```

**2. 成本优化技巧**

| 优化项 | 方法 | 节省比例 |
|--------|------|---------|
| **模型选择** | 简单任务用小模型 | 50-80% |
| **缓存响应** | 相同问题返回缓存 | 30-50% |
| **批量处理** | 合并多个请求 | 20-30% |
| **限制上下文** | 减少 token 使用 | 20-40% |
| **本地模型** | 简单任务本地处理 | 100% |

**3. 预算预警设置**
```yaml
# config.yaml 配置
cost_control:
  daily_limit: 10      # 每日预算（元）
  monthly_limit: 300   # 每月预算（元）
  warning_threshold: 0.8  # 80% 时预警
  action: "pause"     # 超预算时：pause/notify/continue
  notify:
    email: "your@email.com"
    webhook: "https://your-webhook.com"
```
